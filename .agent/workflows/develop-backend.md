description: Implement a backend Jira ticket following an implementation plan
---

1. Read the implementation plan at $ARGUMENTS (path to the .md file in `ai-specs/changes/`).
2. Start a new branch using the ticket ID (e.g., `feature/SCRUM-1-backend`).
3. Implement the changes step-by-step: Validation -> Service -> Controller -> Routes.
4. Write and run tests to ensure 90%+ coverage.
5. Update technical documentation (data model, API spec).
6. Commit changes and push to a new PR.
