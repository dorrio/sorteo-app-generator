# AGENTS.md — `components/sorteo/`

Application-specific components (drawing modes, GEO/SEO sections, visual editor, winner ceremony).

## What lives here

- `main-app.tsx` — the main shell. Orchestrates participants, theme, drawing, and the current `sorteoStyle`. Loaded via `next/dynamic` from pages.
- `modes/` — the six drawing-mode implementations. `sorteoStyle` ∈ `"grid" | "slot-machine" | "roulette" | "cascade" | "cards" | "matrix"`. Each mode owns its animation and consumes `useSorteoStore`.
- `*-geo.tsx` — Direct Answer / GEO (Generative Engine Optimization) blocks for each tool's landing page. **Must stay server-renderable, no `framer-motion`, no `animate-pulse`.**
- `seo-content.tsx`, `glossary.tsx`, `site-footer.tsx` — static SEO sections; same rule.
- `wheel-geo.tsx` + `wheel-visual.tsx` + `try-tool-button.tsx` — example of the "static server parent + small client island" pattern.
- `participant-manager.tsx`, `visual-editor.tsx`, `winner-ceremony.tsx`, `verification-modal.tsx` — heavy interactive client components.
- `hooks/` — local hooks used only by this folder's components.

## Hard rules

1. **Winner selection** always via `selectSecureWinner()` from `@/lib/sorteo-store`. Never `Math.random()`.
2. **`-geo.tsx` components and `seo-content.tsx`, `glossary.tsx`:**
   - Must be **Server Components** (no `"use client"` at the top).
   - No `framer-motion`, no `animate-pulse`, no entrance animations.
   - Extract any interactive bits into small client islands (see `TryToolButton`, `WheelVisual`).
   - Render visible FAQ/HowTo markup AND the matching JSON-LD schema via `safeJsonLdStringify()` — never raw `JSON.stringify` (it doesn't escape `<`, `>`, `&` and will break JSON-LD parsing).
   - Use semantic HTML: `section`, `ul[role="list"]`, `li`, `dl/dt/dd`, `aria-labelledby`. Don't regress into generic `div` soup — `.jules/nexus.md` tracks why.
3. **Heavy client components** (`main-app`, `visual-editor`, `winner-ceremony`, drawing modes, confetti) must be loaded via `next/dynamic` from the page or parent. Don't statically import them from a server component.
4. **Navigation:** import `Link` from `@/i18n/routing`, not `next/link`.
5. **Animations on LCP elements forbidden.** No `initial={{ opacity: 0 }}` on headers, H1s, or the first-paint section.
6. **Event listeners** on scroll/resize/touch must be `{ passive: true }` and throttled via `requestAnimationFrame` (see `sticky-share-footer.tsx`).
7. **Translations:** every user-visible string goes through `useTranslations` and must exist in all three `messages/*.json` files.

## shadcn primitives

`components/ui/*` is generated by the shadcn CLI. Do not hand-edit. To add one: `pnpm dlx shadcn@latest add <component>`. If you need to tweak behavior, wrap the primitive in a new file inside this folder (`components/sorteo/`) rather than patching the generated file.

## Before opening a PR

- `pnpm verify` passes.
- `pnpm build` succeeds (catches RSC boundary violations and i18n key misses).
- If you changed a tool's landing page or any `-geo.tsx`, visit the page in `pnpm dev` and verify the DOM contains the expected schema (`view-source:` on the rendered page) — crawlers must see the markup without JS.
